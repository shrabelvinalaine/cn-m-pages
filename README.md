# cn-m-pages

一个用于归档和发布多个独立 HTML 页面的仓库。  
本项目不针对任何特定域名或网站，仅作为静态 HTML 文件的集中存放与版本管理用途。

## 项目简介

cn-m-pages 旨在以简洁、可维护的方式存放一系列独立的 HTML 页面。  
这些页面可以是单页应用、信息展示页、工具页面或其他形式的静态内容。  
仓库本身不提供导航、搜索或动态服务，仅通过 Git 进行版本控制与发布。

## 目录说明

```
cn-m-pages/
├── pages/          # 存放所有独立 HTML 页面，按需组织子目录
├── assets/         # 公共资源文件（CSS、JS、图片等），可按页面或类型分组
├── README.md       # 本文件
└── .gitignore      # Git 忽略规则
```

- `pages/`：每个 HTML 文件或文件夹代表一个独立页面，命名应见名知义。
- `assets/`：存放页面共用的或页面专用的资源文件，建议保持结构清晰。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，彼此之间无强制依赖关系。  
每个页面可包含自己的样式与脚本，也可引用 `assets/` 下的公共资源。  
归档时请注意：

- 页面文件名应具有描述性，避免使用无意义的名称。
- 如页面需要外部资源，请尽量使用相对路径引用仓库内资源。
- 不建议在页面中嵌入对其他外部服务的硬编码链接。

## 维护说明

- 欢迎通过 Pull Request 提交新的页面归档或改进现有页面。
- 提交前请确保页面能直接通过浏览器打开，并检查无功能错误。
- 请勿在页面中放置恶意内容、跟踪代码或侵犯他人权益的信息。
- 仓库维护者保留对内容进行整理、合并或移除的权利。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/cn-m-pages.git
   ```
2. 在 `pages/` 下添加或修改 HTML 文件。
3. 使用浏览器直接打开本地 HTML 文件预览效果。
4. 提交更改并推送，GitHub Pages 或其他静态托管服务可自动发布。
